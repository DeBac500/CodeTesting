Project guidelines

Development guidelines

  1. All code must be Java 1.3 compliant (since it is the most widely used Java version for now).
  2. Every published class must be accurately commented with javadoc comments.

Documentation guidelines

  1. All documents are written in english
  2. All documents are published in HTML format

Release Procedure

  1. When all parts of the project are working properly locally
  2. Verify and prepare documentation
  3. Check project plan for incomplete required tasks and update status
  4. Commit all code to the source repository
  5. Prepare a delivery package locally
  6. Test the delivery package (when possible on another machine)
  7. Version the Code in the source repository
  8. Update the change log
  9. Prepare screenshots for the web
  10. Update news and web locally
  11. Release the delivery package in zip format
  12. Release the web site content
